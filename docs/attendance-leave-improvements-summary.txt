SmartRAD 근태·휴가 관리 기능 개선 작업 공유

브랜치
feature/attendance-leave-improvements

1. 작업 개요

휴가 신청 편의성과 관리자 근태 통계의 정확도를 개선했습니다.

주요 작업 내용
- 승인 대기 휴가 신청 취소
- 휴가 신청 일수 및 신청 후 잔여 일수 미리보기
- 관리자 근태 통계의 재직자 기준 통일
- 근태·휴가 페이지 헤더 액션 버튼 개선

참고
- 근태 정정 요청 및 관리자 정정 관리 기능은 최종 작업 범위에서 제외했습니다.
- 기존 근태 사유 입력·수정 기능은 그대로 유지됩니다.

--------------------------------------------------

2. 휴가 신청 취소 기능

기능 설명
- 직원이 /leave/my 페이지에서 본인의 승인 대기 중인 휴가 신청을 취소할 수 있습니다.

처리 조건
- 로그인한 직원 본인의 신청만 취소할 수 있습니다.
- PENDING 상태의 신청만 취소할 수 있습니다.
- APPROVED, REJECTED, CANCELLED 상태의 신청은 취소할 수 없습니다.
- 다른 직원의 신청에는 접근할 수 없습니다.
- 존재하지 않는 신청은 기존 Not Found 예외 형식으로 처리됩니다.

추가 상태
CANCELLED

취소 시 변경 사항
- status를 CANCELLED로 변경
- processedAt에 취소 처리 시각 기록

추가 API
PATCH /api/leave-requests/me/{leaveRequestId}/cancel

프론트엔드 변경
- /leave/my 상태 필터에 "신청 취소" 추가
- 취소 상태에 회색 계열 배지 적용
- PENDING 상태의 상세 모달에만 "신청 취소" 버튼 표시
- 취소 전 공통 확인 모달 표시
- 취소 처리 중 버튼 비활성화
- 취소 성공 후 상세 모달 닫기
- 휴가 신청 내역, 요약 카드, 잔여 일수 재조회
- 취소 완료 안내 메시지 표시

집계 처리
- 취소된 신청은 승인 대기 휴가 일수에서 제외됩니다.
- 취소된 신청은 휴가 사용 일수에서 제외됩니다.
- 취소된 신청은 관리자 승인 대상에서 제외됩니다.
- 취소된 신청은 휴가 기간 중복 검사에서 제외됩니다.

--------------------------------------------------

3. 휴가 신청 일수 미리보기

기능 설명
- 휴가 유형, 시작일, 종료일을 입력하면 신청 전에 실제 사용 일수와 예상 잔여 일수를 확인할 수 있습니다.

추가 API
POST /api/leave-requests/me/preview
Content-Type: application/json

요청 예시
{
  "leaveTypeId": 1,
  "startDate": "2026-07-23",
  "endDate": "2026-07-27"
}

응답 예시
{
  "leaveTypeId": 1,
  "leaveTypeName": "연차",
  "startDate": "2026-07-23",
  "endDate": "2026-07-27",
  "requestedDays": 3,
  "currentRemainDays": 7,
  "remainDaysAfterRequest": 4,
  "available": true,
  "message": null
}

일수 계산 기준
- 시작일과 종료일을 모두 포함합니다.
- 토요일과 일요일은 신청 일수에서 제외합니다.
- 시작일이 종료일보다 늦으면 신청할 수 없습니다.
- 주말만 포함되어 실제 신청 일수가 0일이면 신청할 수 없습니다.
- PENDING, APPROVED 상태의 기존 휴가와 기간이 중복되면 신청할 수 없습니다.
- REJECTED, CANCELLED 상태는 중복 검사에서 제외됩니다.
- 신청 일수가 현재 잔여 일수보다 크면 신청할 수 없습니다.

공통 계산 로직
- 미리보기와 실제 신청이 동일한 백엔드 계산 로직을 사용합니다.
- 실제 제출 시 서버에서 기간, 일수, 중복 여부와 잔여 일수를 다시 검증합니다.
- 프론트 미리보기 결과만 신뢰하여 휴가를 등록하지 않습니다.

프론트엔드 변경
- 주말 제외 실제 사용 일수 표시
- 현재 잔여 일수 표시
- 신청 후 예상 잔여 일수 표시
- 중복 신청 또는 잔여 일수 부족 안내 표시
- 미리보기 조회 중 로딩 표시
- 입력 변경 시 약 350ms 디바운스 적용

신청 버튼 비활성화 조건
- 미리보기 조회 중
- 미리보기 결과가 없는 경우
- 기간이 중복된 경우
- 잔여 일수가 부족한 경우
- 서버 검증 결과 신청할 수 없는 경우

4. 관리자 근태 통계의 재직자 기준 통일

변경 배경
- 기존 관리자 근태 화면에서 전체 직원 수와 재직 직원 수가 혼용되어 통계마다 기준 인원이 다르게 표시될 수 있었습니다.

통일 기준
employee.employeeStatusCode === "ACTIVE"

적용 영역
- 일일 근태 현황의 전체 대상 직원 수
- 출근 및 미출근 인원 계산
- 관리자 사이드 패널 전체 인원
- 수동 근태 등록 대상
- 월간 근태 통계 대상 직원
- 월간 직원별 근태 요약
- 부서별 근태 통계
- 근태 경고 대상 직원
- 월간 CSV 내보내기 대상

변경 결과
- 퇴직 직원은 현재 근태 통계에서 제외됩니다.
- 일일 통계와 월간 통계가 동일한 직원 기준을 사용합니다.
- 전체 인원과 출근·미출근 인원의 모수가 일치합니다.
- 필터링된 ACTIVE 직원 목록을 통계 계산에 공통으로 사용합니다.

5. 페이지 헤더 액션 버튼 개선

/attendance/self
- "오늘 기록 새로고침" 문구를 "새로고침"으로 변경했습니다.
- 기존 새로고침 동작은 그대로 유지합니다.
- 흰색 보조 버튼을 보라색 주요 액션 버튼으로 변경했습니다.

/attendance/my
- 조회 기간 영역에 있던 새로고침 버튼을 제거했습니다.
- 동일한 버튼을 공통 페이지 헤더 우측으로 이동했습니다.
- 기존 데이터 재조회 함수는 그대로 유지합니다.
- 보라색 주요 액션 스타일을 적용했습니다.

/leave/my
- 본문 상단에 있던 휴가 신청 버튼을 제거했습니다.
- 동일한 버튼을 공통 페이지 헤더 우측으로 이동했습니다.
- 기존 휴가 신청 모달 호출 함수는 그대로 유지합니다.
- 버튼 제거 후 남는 불필요한 본문 여백을 정리했습니다.

반응형 처리
- 데스크톱에서는 제목과 경로 정보를 헤더 왼쪽에 유지합니다.
- 액션 버튼은 헤더 오른쪽에 정렬합니다.
- 모바일에서는 액션 영역이 제목 아래로 내려갑니다.

7. 기존 기능 보호

다음 기존 기능은 삭제하거나 변경하지 않았습니다.

- 출근 및 퇴근 처리
- 개인 근태 조회
- 기존 근태 사유 입력·수정
- 근태 증빙 파일 처리
- 관리자 일일 근태 조회
- 관리자 수동 근태 등록
- 기존 휴가 신청
- 휴가 승인·반려
- 휴가 잔여 일수 차감
- 휴가 사용 현황
- 기존 인증 및 권한 처리
- 기존 알림 기능

근태 정정 요청 및 관리자 승인·반려 기능은 작업 중 검토했으나 최종 범위에서 관련 프론트와 백엔드 구현을 모두 제거했습니다.

8. 검증 결과

통과 항목
- 백엔드 compileJava 성공
- 변경된 프론트엔드 파일 ESLint 통과
- git diff --check 통과
- 근태 정정 기능 관련 파일 및 참조 제거 확인

기존 프로젝트 이슈
- 전체 프론트엔드 빌드는 react-organizational-chart 의존성 누락으로 중단됩니다.
- 백엔드 전체 테스트는 기존 테스트 환경의 DB dialect 설정 문제로 contextLoads가 실패합니다.
- 위 두 문제는 이번 휴가 및 근태 통계 변경에서 발생한 오류가 아닙니다.
